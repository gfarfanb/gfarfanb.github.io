Jenkins en Ubuntu 16.04

01 Oct 2016 . categoría: compilation . Comentarios
#ci #cd #ubuntu #jenkins

Si necesitas a alguien que se encargue por ti en las tareas de ciclo de vida tales como construcción, documentación, pruebas, empaquetado, despliegue, análisis estático o alguna otra, simplemente pon a trabajar a un Jenkins. Hablando técnicamente es uno de los mejores sistemas que he conocido por su manera en que está construido y por su funcionalidad. Desde mi punto de vista es pionero en esto de la Integración Continua y ahora con un soporte más evidente para el Despliegue Continuo. Permite la gestión de todo lo que tiene que ver con ambiente, cuentas, extensiones, distribución del sistema, seguridad, tiene además una comunidad muy grande y un universo de extensiones (¿alguna idea que tengas de automatización? da por hecho que podrás encontrar el plugin respectivo). La imaginación para todo lo que puedas automatizar ya depende de ti.

En este Post se mostrará instalación y configuración inicial de Jenkins. Así como tambíen los apartados para personalizar el sistema, integrar complementos o herramientas, y una introducción de cómo repartir carga de trabajo de Jenkins a máquinas externas (Jenkins distribuido).

 

Prerrequisitos

  1. Generación de usuario Jenkins (no asignes privilegios de administrador).
  2. Generación de App Password para uso de Gmail como servidor SMTP, [1].
Importante: Ambiente
Ubuntu 16.04.1 LTS alojado en DigitalOcean.

 

Instalación de Jenkins

Dentro de la opción de descarga del sitio de Jenkins se encuentra el listado de todos los sistemas sobre los que lo puedes instalar, en nuestro caso seleccionamos Ubuntu/Debian [2].

  • Necesitas una llave en tu Ubuntu para el repositorio de Jenkins, que te ayude a que la instalación y actualización sea automática.

wget -q -O - http://pkg.jenkins.io/debian/jenkins.io.key | sudo apt-key add -


  • Agrega la siguiente entrada en tu archivo de sistema /etc/apt/sources.list.

sudo vim /etc/apt/sources.list


# Última línea de archivo /etc/apt/sources.list
deb http://pkg.jenkins.io/debian binary/


  • Posteriormente podrás obtener el paquete Jenkins de APT.

sudo apt-get update
sudo apt-get install jenkins

 

Iniciando con Jenkins

Una vez instalado el servidor de Jenkins este se encontrará disponible en el puerto 8080 (puerto default).

Importante:
Necesitas ser administrador para realizar esta operación.
  • Ingresa por primera vez al servidor (http://localhost:8080 o http://dominio_de_servidor_o_IP:8080), aparecerá una pantalla para desbloquear Jenkins. Posteriormente, copia desde el servidor el contenido del archivo /var/lib/jenkins/secrets/initialAdminPassword y pégalo en el campo Administrator password de la pantalla inicial de Jenkins.

sudo cat /var/lib/jenkins/secrets/initialAdminPassword


  • Confirmada la contraseña de administrador aparecen dos opciones para instalación de plugins: sugeridos o más utilizados y elección libre. Para efectos prácticos seleccionamos Install suggested plugins (de cualquier manera tu podrás instalar o desinslatar plugins a tu gusto con el Manage Plugins de Jenkins). Automáticamente se realizará la instalación de los plugins.

  • Para finalizar aparecerá un formulario con el que podrás generar un usuario Admin para Jenkins. También es posible continuar como administrador.

  • Tu Jenkins está listo, ya podrás accesar a tu panel de control y empezar a generar tareas, instalar más plugins, conectar nodos, explotar la funcionalidad de tu Jenkins.

 

Configuración de sistema

El acceso a la configuración del sistema es en Home > Manage Jenkins > Configure System. Aquí puedes gestionar la instalación del sistema, variables de entorno, formatos de fechas, direcciones y plantillas de correo, configuración de servidores, configuración de plugins, entre otras. Describiré aquellas relevantes para la instalación inicial de Jenkins.

Tip:
Prácticamente todos los campos para entradas de usuario tienen un boton de ayuda en el que se describe la razón del campo.

El primer apartado contiene información sobre dónde se encuentra instalado físicamente Jenkins, así como las ubicaciones para espacios de trabajo y registro de ejecuciones. Dentro del mismo apartado puedes configurar el número de ejecutores para el nodo principal (la decisión de cuántos depende del diseño de tus tareas, pero definitivamente los 2 por default no suelen ser suficientes). También puedes encontrar la política de utilización del nodo principal, el tiempo de espera entre ejecuciones, reintentos de peticiones a repositorio, por mencionar algunas.

Tip:
Particularmente solo suelo configurar el número de ejecutores y personalizo el mensaje del sistema.

 

Jenkins Location

Básicamente se configura la dirección Web donde puedes acceder a Jenkins, es prácticamente una URL base para que Jenkins pueda construir enlaces (si tienes un dominio asignado para Jenkins necesariamente tienes que configurar este apartado). Por otra parte estableces la dirección de Email para que los correos que sean enviados desde Jenkins tengan un remitente apropiado.

 

Git plugin

Este se utiliza para integrar Git con Jenkins, en este apartado se configura la identidad de los commits cuando sean generados por Jenkins.

 

Notificaciones por correo

No tendría mucho sentido tener un Ingeniero de Integración y Despliegue si no pudiera avisarte por lo menos que ha terminado su trabajo. Las notificaciones por correo es una de las tantas alternativas para notificación del estatus de las tareas ejecutadas en Jenkins.

Nota:
Puedes utilizar el servidor SMTP de tu agrado, por si no quieres generar un App Password para uso de Gmail como servidor SMTP.

 

Caja de herramientas

Podemos clasificar la herramientas que utilizas en Jenkins de la siguiente manera:

a. Internas o aquellas que se integran como complemento a Jenkins. Necesitan de la instalación/configuración de un plugin y de la instalación, en el servidor, de la distribución a la que hace referencia el plugin. Las herramientas Internas las configuras en Home > Manage Jenkins > Global Tool Configuration.

b. Externas o aquellas que utilizas mediante línea de comandos desde Jenkins, que se encuentran instaladas en el servidor. No necesitas un plugin pero si necesitas conocer perfectamente los Paths de las distribuciones.

Importante:
Considera que Jenkins se puede distribuir mediante nodos, así que la gestión de Paths debe ser por nodo.
Nota:
Prácticamente cualquier distribución que utilices para una herramienta Interna podrás utilizarla como herramienta Externa.

 

JDK

Para realizar compilaciones de aplicaciones Java necesitarás instalar el JDK independientemente de la herramienta de construcción que ocupes para ello [3]. Para nuestra configuración instalaremos JDK 8.

  • Instala el JDK por default (el JAVA_HOME para esta instalación será /usr/lib/jvm/java-8-openjdk-amd64).

sudo apt-get install default-jdk


  • O si prefieres estar consciente de que instalas la versión apropiada (el JAVA_HOME para esta instalación será /usr/lib/jvm/java-8-oracle).

sudo apt-get install python-software-properties
sudo add-apt-repository ppa:webupd8team/java
sudo apt-get update


sudo apt-get install oracle-java8-installer
Tip:
Si tienes más de una instalación de Java puedes administrarlas con el siguiente comando: sudo update-alternatives --config javac


  • Agrega una instalación en Global Tool Configuration dentro de JDK. Especifica un nombre (el de la versión de JDK sería el más indicado) y el Path del JAVA_HOME.

 

Git y SSH/GitHub

  • Primero verifica que se encuentre instalado Git. Ya sea que lo tengas que instalar o solo configurar puedes seguir la siguiente referencia: Instalación de Git.

git --version


  • En Global Tool Configuration en el apartado para Git, agrega una instalación. Unicamente se define un nombre y le introduce el Path del ejecutable de Git.
Tip:
Si el comando git se encuentra disponible en el sistema, no necesitas especificar todo el Path del ejecutable. A este tipo de configuración se designa por convensión como Default.

 

Gradle

Construcción de multi-proyectos, incremental, resolución de dependencias, ejecución de pruebas. Una herramienta con lo necesario y todavía más para poder automatizar tareas de integración de tus proyectos como productos.

Nota:
Si cuentas con scripts de Maven o Ant y no requieres re-escribirlos, puedes fácilmente incluirlos a tu script Gradle.
  • Instala Gradle agregando los repositorios respectivos. Por default se instalará la versión estable más actual (el GRADLE_HOME para la instalación será parecido a /usr/lib/gradle/3.x) [4].

sudo add-apt-repository ppa:cwchien/gradle
sudo apt-get update


sudo apt-get install gradle


  • En Global Tool Configuration en el apartado para Gradle, agrega un nombre para tu instalación y el Path del GRADLE_HOME.

 

Plugins para tu Jenkinstain

La administración de plugins el muy sencilla. Para empezar, Jenkins detecta las actualizaciones (no solo de plugins, sino del sistema en general). Todas las actualizaciones disponibles se encuentran en la vista principal de Home > Manage Jenkins > Manage Plugins. Solo debes seleccionar aquellos que requieres actualizar.

Tip:
De vez en cuando ingresa a Manage Jenkins y verás alertas de lo que necesitas administrar de Jenkins.

Todos los plugins que seleccionaste y una vez que diste la opción Descargar ahora e instalar después de reiniciar se descargarán e instalarán de manera automática. Las actualizaciones las podrás ver después de que tu Jenkins sea reiniciado (manual, automáticamente o drásticamente).

Por otra parte, puedes visualizar todos los plugins oficiales de Jenkins (con la opción de instalar) y todos los plugins que tienes instalados en tu sistema (con la opción de desinstalar). El filtro de búsqueda es muy útil cuando el listado de plugins, independientemente del apartado, sea muy extenso.

En algunos casos necesitas tener acceso a Internet mediante Proxy. Para ello cuentas con un apartado propio para dicha configuración. El formulario es muy básico, pero te permite capturar los Hosts de los que no requieras interferencia de Proxy, además de que en la opción avanzada puedes realizar una prueba de tu configuración.

Existen ocasiones en las que conoces un plugin útil pero no se encuentra listado de los plugins oficiales de Jenkins, o simplemente cuentas con el archivo HPI. Puedes en ese caso, instalar o actualizar tu plugin de forma manual. ¡Sube el archivo y listo!

Hablando de la instalación automática, por default Jenkins tiene establecida una especificación que le permite identificar las actualizaciones y conocer la ubicación para descarga de cada uno de los plugins (muy simple el mecanismo pero muy efectivo). Si tu cuentas con una especificación que haga referencia a plugins particulares, oficiales o no oficiales, puedes cambiar la URL y llevar el control de las actualizaciones de plugins con una especificación de tu elección.

 

Nodos para distribuir Jenkins

Al darte Jenkins la posibilidad de integrar diferentes tecnologías, surgen algunas questiones ¿Todas las debo instalar en mi servidor? ¿Qué va a pasar con el rendimiento? ¿Y si no encuentro las versiones apropiadas para el sistema operativo de mi servidor? Todas esas preguntas tienen solución: distribuyendo Jenkins.

Jenkins tiene la opción de administrar nodos (o esclavos) [5]. Y por nodos entiendanse todas aquellas máquinas virtuales o físicas (independiente de sistema operativo) de las que podrá disponer Jenkins para dividir la ejecución de sus tareas. ¡Es correcto! puedes tener un servidor Jenkins con los requerimientos mínimos del sistema y aun así ejecutar procesos de días de ejecución (aunque suene exagerado).

Nota:
EL propio servidor de Jenkins es también un nodo (considerado como principal o maestro).

Los nodos cuentan con algunas propiedades interesantes como: variables de entorno únicas, control sobre el número de ejecutores, reglas para su frecuencia de utilización y disponibilidad, comunicación para compartir contenido (entre nodos), mecanismos de monitorización (arquitectura, reloj, espacio en disco, timeout), por mencionar las más importantes.

Para administrar nodos ingresa a Home > Manage Jenkins > Manage Nodes. Ahí encontrarás el estado de los nodos que tienes asociados y las opciones de creación de nodos y de configuración de monitorización.

  • Selecciona New Node para registrar un nuevo nodo. Define un nombre y selecciona el tipo.

  • Especifica la cantidad de ejecutores que utilizarás y el directorio donde se instalará el binario de tu nodo. Si necesitas organizar tus nodos, puedes establecer etiquetas. Por otra parte debes seleccionar el tipo de uso, el mecanismo para ejecutar remotamente tu nodo y el criterio de disponibilidad.

  • A continuación se muestran las configuraciones para un nodo Unix y un nodo Windows, en ambos casos el último paso es Save.

 

Configuración de Nodo Unix con SSH

  • Necesitas preparar tu nodo para poder realizar una conexión por SSH [6]:
  1. Genera el usuario del nodo.
  2. Si no cuentas con un servidor SSH, realiza la instalación.
  3. Asignar permisos para conexión por SSH del nuevo usuario.
  4. Conectado con el nuevo usuario, debes generar un directorio ~/.ssh con permisos de lectura y escritura.
  5. Genera tu llave SSH.
  6. Asigna tu llave pública al registro de llaves autorizadas.
  • En el campo Launch method selecciona la opción Launch slave agents on Unix machines via SSH. Posteriormente ingresa un dominio_de_servidor_o_IP y selecciona una credencial de tipo usuario/contraseña. Si no cuentas con una credencial selecciona Add -> Jenkins para configurar una credencial y completar tu configuración.

 

Configuración de Nodo Windows

  • Aunque es posible configurar un nodo Windows con SSH o mediante acceso por red, resulta más práctico utilizar el mecanismo JNLP. En este sentido, tenemos que habilitar el puerto TCP para JNLP accesando a Home > Manage Jenkins > Configure Global Security. Si conoces o necesitas de algún puerto en particular defínelo como Fixed o establécelo aleatorio Random. Guarda los cambios.

  • Volviendo a la configuración del nodo (Home > Manage Jenkins > Manage Nodes > Nombre_de_nodo), selecciona la opción Launch slave agents via Java Web Start para el campo Launch method.

  • Después de guardar ocurrirá una conexión fallida (es completamente normal). Selecciona el nodo y aparecerá una opción Launch que realizará la descarga de un artefacto slave-agent.jnlp.

  • Ejecuta el artefacto slave-agent.jnlp, te pedirá confirmación. Una vez conectado puedes instalar el nodo como servicio File > Install as Windows Service para no tener que repetir esta operación cada que necesites del nodo.

 

Conclusión

Jenkins te da la opción de poder gestionar procesos de integración y despliegue de muchas tecnologías. Es muy recomendable para aquellos que prefieren ser los autores de su propios diseños de automatización. Es una instalación sencilla y los conceptos se pueden ir aprendiendo conforme se vaya utilizando el sistema.
